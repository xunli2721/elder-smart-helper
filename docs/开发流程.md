# ElderSmartHelper 开发流程

本文档记录项目的完整开发步骤，按顺序执行即可完成整个项目。

---

## 第一步：精简数据库 Schema

**目标**：将 8 张表精简为 4 张核心表

**保留的表**：
- `users` — 用户信息（老人、家人、管理员）
- `family_relationships` — 家庭绑定关系
- `tutorials` — 教程内容（图文步骤）
- `remote_sessions` — 远程协助会话

**去掉的表**：
- `devices`、`security_events`、`user_activity_log`、`app_settings`、`migration_history`

**涉及文件**：`database/schema.sql`

---

## 第二步：搭建后端基础框架

**目标**：创建 Express 服务器，连接 MySQL，去除 Redis 依赖

**工作内容**：
- 精简 `server/package.json`，去掉不需要的依赖
- 创建 `server/src/config/db.js` — MySQL 连接配置
- 精简 `server/src/index.js` — 去掉 Redis、Helmet、Morgan
- 创建 `.env` 文件

**涉及文件**：
- `server/package.json`
- `server/src/index.js`
- `server/src/config/db.js`
- `server/.env`

---

## 第三步：实现用户认证 API

**目标**：注册、登录、获取用户信息

**API 接口**：
- `POST /api/auth/register` — 注册（手机号、密码、姓名、用户类型）
- `POST /api/auth/login` — 登录（返回 JWT）
- `GET /api/auth/profile` — 获取当前用户信息（需认证）

**涉及文件**：
- `server/src/middleware/auth.js` — JWT 验证中间件
- `server/src/routes/auth.routes.js` — 认证路由
- `server/src/controllers/auth.controller.js` — 认证逻辑

---

## 第四步：实现教程 API

**目标**：教程的增删改查

**API 接口**：
- `GET /api/tutorials` — 获取教程列表（支持分类筛选）
- `GET /api/tutorials/:id` — 获取教程详情
- `POST /api/tutorials` — 创建教程（管理员）
- `PUT /api/tutorials/:id` — 更新教程（管理员）
- `DELETE /api/tutorials/:id` — 删除教程（管理员）

**涉及文件**：
- `server/src/routes/tutorial.routes.js`
- `server/src/controllers/tutorial.controller.js`

---

## 第五步：实现家人绑定 API

**目标**：老人与家人建立绑定关系

**API 接口**：
- `POST /api/users/bind` — 通过手机号绑定家人
- `GET /api/users/family` — 获取已绑定的家人列表
- `DELETE /api/users/family/:id` — 解除绑定

**涉及文件**：
- `server/src/routes/user.routes.js`
- `server/src/controllers/user.controller.js`

---

## 第六步：实现远程协助 API + WebSocket

**目标**：远程协助的会话管理和实时通信

**API 接口**：
- `POST /api/remote/request` — 发起协助请求
- `GET /api/remote/sessions` — 获取会话列表
- `PUT /api/remote/sessions/:id/status` — 更新会话状态

**WebSocket 事件**：
- `screenshot` — 老人端发送截图
- `annotation` — 家人端发送标注（画圈/箭头）
- `message` — 文字消息
- `end_session` — 结束会话

**涉及文件**：
- `server/src/routes/remote.routes.js`
- `server/src/controllers/remote.controller.js`
- `server/src/services/socketService.js`

---

## 第七步：创建 Flutter 项目

**目标**：初始化 Flutter 项目，配置主题和导航

**工作内容**：
- `flutter create flutter_app`
- 配置适老化主题（大字体、高对比度颜色）
- 配置底部导航栏（首页、教程、远程协助、设置）
- 创建页面路由

**涉及文件**：
- `flutter_app/lib/main.dart`
- `flutter_app/lib/config/theme.dart`
- `flutter_app/lib/config/routes.dart`
- `flutter_app/pubspec.yaml`

---

## 第八步：实现登录/注册页面

**目标**：用户可以注册和登录

**页面**：
- 登录页：手机号 + 密码，登录按钮
- 注册页：手机号 + 密码 + 姓名 + 用户类型（老人/家人）

**涉及文件**：
- `flutter_app/lib/screens/login_screen.dart`
- `flutter_app/lib/screens/register_screen.dart`
- `flutter_app/lib/services/api_service.dart`
- `flutter_app/lib/models/user.dart`

---

## 第九步：实现首页

**内容**：
- 顶部问候语

- `flutter_app/lib/screens/home_screen.dart`

---

## 第十步：实现教程页面

**目标**：教程列表和详情展示

**页面**：
- 教程列表页：按分类筛选，卡片展示
- 教程详情页：分步骤展示图文内容，大按钮翻页

**涉及文件**：
- `flutter_app/lib/screens/tutorial_list_screen.dart`
- `flutter_app/lib/screens/tutorial_detail_screen.dart`

---

## 第十一步：实现远程协助页面

**目标**：老人发起求助，家人标注指导

**功能**：
- 老人端：发起协助请求 → 截屏发送 → 查看家人标注 → 接收文字指导
- 家人端：收到请求 → 查看截图 → 画圈/箭头标注 → 发送文字消息

**涉及文件**：
- `flutter_app/lib/screens/remote_assist_screen.dart`
- `flutter_app/lib/services/socket_service.dart`
- `flutter_app/lib/widgets/annotation_canvas.dart`

---

## 第十二步：实现设置页面

**目标**：个人信息和字体大小设置

**内容**：
- 个人信息展示与编辑
- 字体大小调节（小/中/大/超大，实时预览）
- 退出登录

**涉及文件**：
- `flutter_app/lib/screens/settings_screen.dart`

---

## 第十三步：联调与测试

**目标**：前后端联调，确保主流程可用

**测试流程**：
1. 注册老人账号和家人账号
2. 家人绑定老人
3. 老人浏览教程
4. 老人发起远程协助
5. 家人接收请求并标注指导
6. 结束协助

**验证方式**：
- 后端：`npm run dev` 启动，用 curl 或 Postman 测试 API
- 前端：`flutter run` 在 Android 模拟器/真机上运行
- 联调：确保 Flutter 能正确调用后端 API
