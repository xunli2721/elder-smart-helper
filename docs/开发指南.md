# ElderSmartHelper 开发指南

## 开发环境搭建

### 系统要求
- **操作系统**：Windows 10/11, macOS 10.15+, Ubuntu 20.04+
- **Node.js**：v16.0.0 或更高版本
- **Python**：3.8 或更高版本
- **数据库**：MySQL 8.0+, Redis 6.0+
- **移动开发**：Android Studio（Android开发）, Xcode（iOS开发，仅macOS）

### 环境配置步骤

#### 1. 克隆项目
```bash
git clone https://gitee.com/lzbaawso/elder-smart-helper.git
cd elder-smart-helper
```

#### 2. 后端环境配置
```bash
# 进入后端目录
cd server

# 安装依赖
npm install

# 配置环境变量
cp .env.example .env
# 编辑 .env 文件，设置数据库连接等信息

# 启动数据库服务
# MySQL 和 Redis 需要提前安装并启动

# 初始化数据库
mysql -u root -p < ../database/schema.sql

# 启动开发服务器
npm run dev
```

#### 3. 移动端环境配置
```bash
# 进入移动端目录
cd mobile-app

# 安装依赖
npm install

# 配置环境变量
cp .env.example .env
# 编辑 .env 文件，设置API地址等

# iOS 额外配置（仅macOS）
cd ios && pod install && cd ..

# 启动开发服务器
npm start

# 新开终端，运行应用
npm run android  # 或 npm run ios
```

## 编码规范

### 通用规范
1. **命名约定**
   - 变量/函数：camelCase
   - 类名/组件：PascalCase
   - 常量：UPPER_SNAKE_CASE
   - 文件命名：kebab-case

2. **代码风格**
   - 使用ES6+语法
   - 避免使用var，使用const/let
   - 箭头函数优先
   - 模板字符串替代字符串拼接

3. **注释规范**
   - 函数注释：说明功能、参数、返回值
   - 复杂逻辑：添加行内注释
   - TODO/FIXME：标记待办事项

### React Native 规范
1. **组件规范**
   ```javascript
   // 函数组件示例
   import React from 'react';
   import { View, Text, StyleSheet } from 'react-native';
   import PropTypes from 'prop-types';

   const MyComponent = ({ title, onPress }) => {
     return (
       <View style={styles.container}>
         <Text style={styles.title}>{title}</Text>
       </View>
     );
   };

   MyComponent.propTypes = {
     title: PropTypes.string.isRequired,
     onPress: PropTypes.func,
   };

   MyComponent.defaultProps = {
     onPress: () => {},
   };

   const styles = StyleSheet.create({
     container: {
       padding: 16,
       backgroundColor: '#fff',
     },
     title: {
       fontSize: 18,
       fontWeight: 'bold',
     },
   });

   export default MyComponent;
   ```

2. **样式规范**
   - 使用StyleSheet.create创建样式
   - 优先使用Flex布局
   - 为适老化设计：最小点击区域44x44px
   - 颜色使用主题系统

### 后端规范
1. **路由规范**
   ```javascript
   // routes/user.routes.js
   const express = require('express');
   const router = express.Router();
   const userController = require('../controllers/user.controller');
   const authMiddleware = require('../middleware/auth');

   // 公共路由
   router.post('/register', userController.register);
   router.post('/login', userController.login);

   // 需要认证的路由
   router.use(authMiddleware.verifyToken);
   router.get('/profile', userController.getProfile);
   router.put('/profile', userController.updateProfile);

   module.exports = router;
   ```

2. **错误处理**
   ```javascript
   // middleware/errorHandler.js
   const errorHandler = (err, req, res, next) => {
     console.error('Error:', err);

     const statusCode = err.statusCode || 500;
     const message = err.message || 'Internal Server Error';

     res.status(statusCode).json({
       success: false,
       error: {
         code: err.code || 'INTERNAL_ERROR',
         message: message,
         details: process.env.NODE_ENV === 'development' ? err.stack : undefined,
       },
     });
   };
   ```

## 适老化设计指南

### 视觉设计原则
1. **字体大小**
   - 正文：最小18sp（Android）/18pt（iOS）
   - 标题：24sp/pt 以上
   - 支持动态字体大小调整

2. **颜色对比**
   - 文本与背景对比度 ≥ 4.5:1
   - 重要元素使用高对比色
   - 提供高对比度模式

3. **触摸目标**
   - 最小尺寸：44x44px
   - 元素间间距：8px 以上
   - 避免密集排列

### 交互设计原则
1. **操作简化**
   - 一步完成一个任务
   - 减少操作层级
   - 提供快捷方式

2. **反馈明确**
   - 操作后有视觉/声音反馈
   - 加载状态清晰可见
   - 错误提示友好易懂

3. **容错设计**
   - 支持撤销操作
   - 提供操作确认
   - 自动保存进度

### 无障碍支持
1. **屏幕阅读器**
   - 为所有控件添加accessibilityLabel
   - 提供有意义的accessibilityHint
   - 测试VoiceOver/TalkBack兼容性

2. **键盘导航**
   - 支持键盘Tab键导航
   - 焦点状态清晰可见
   - 提供键盘快捷键

### 单元测试
```javascript

const userService = require('../src/services/user.service');

describe('User Service', () => {
  test('should create user successfully', async () => {
    const userData = {
      phone: '13800138000',
      name: '张三',
      userType: 'elderly',
    };

    const user = await userService.createUser(userData);
    expect(user).toHaveProperty('id');
    expect(user.phone).toBe(userData.phone);
  });
});
```

```javascript
import React from 'react';
import { render, fireEvent } from '@testing-library/react-native';
import Button from '../src/components/Button';

describe('Button Component', () => {
  test('renders correctly', () => {
    const { getByText } = render(<Button title="测试按钮" />);
    expect(getByText('测试按钮')).toBeTruthy();

  test('handles press event', () => {
    const onPress = jest.fn();

    expect(onPress).toHaveBeenCalled();
  });
});
```

1. **生产环境配置**
   ```bash

   export NODE_ENV=production
   export DB_HOST=production-db.example.com
   export REDIS_HOST=production-redis.example.com

   # 安装生产依赖
   npm ci --only=production

   npm start
   ```

   ```bash
   pm2 startup
   ```

## 常见问题

### 开发环境问题
1. **Node版本冲突**
   ```bash
   # 使用nvm管理Node版本
   nvm install 16
   nvm use 16
   ```

2. **iOS模拟器无法启动**
   ```bash
   # 重置模拟器
   xcrun simctl shutdown all
   xcrun simctl erase all
   ```

3. **Android构建失败**
   ```bash
   # 清理缓存
   cd android && ./gradlew clean
   cd .. && npm start --reset-cache
   ```

### 运行时问题
1. **数据库连接失败**
   - 检查数据库服务是否运行
   - 验证连接配置
   - 检查防火墙设置

2. **API请求超时**
   - 检查网络连接
   - 验证API地址配置
   - 查看服务端日志

3. **内存泄漏**
   - 使用Chrome DevTools分析
   - 检查事件监听器清理
   - 优化图片加载
